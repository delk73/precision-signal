Dual-board observer timing evidence packet.

Status:
Capture not retained in this checkout. The execution environment has no visible
observer serial device under /dev/ttyACM*, /dev/ttyUSB*, or /dev/serial, so the
generated files must be produced on the hardware bench.

Artifact boundary:
Do not edit generated files after capture:
timing_report.txt
meta.json
wiring.txt

Generated files:
timing_report.txt: raw SYNC_TIMING_CAPTURE_V1 report from Board B observer.
meta.json: observer-profile metadata generated by scripts/hil_timing_capture.py.
wiring.txt: observer-profile wiring generated for dual_edge_timing_observer_v1.

Manual/topology files:
run_context.json: dual-board topology and flash-before-capture procedure.
notes.txt: operator context, board identity, serial path, reset order, and result
classification.

Topology:
Board A is the actor and runs single_board_tim2_hardware_ack_v1 behavior.
Board B is the observer and runs dual_edge_timing_observer_v1 behavior.

Required wiring:
Board A PA6/D12 -> Board A PA0/A0
Board A PA6/D12 -> Board B PB8/TIM4_CH3
Board A PA1/A1 -> Board B PB9/TIM4_CH4
Board A GND -> Board B GND

Flash-before-capture sequence:
1. Flash Board B observer:
   FW_FEATURES="sync_timing_observer" make flash-ur

2. Flash Board A actor:
   FW_FEATURES="sync_trigger_out sync_trigger_in sync_timing_capture" make flash-ur

3. Capture from Board B serial:
   python3 scripts/hil_timing_capture.py \
     --profile dual_edge_timing_observer_v1 \
     --serial <observer_serial> \
     --out artifacts/hil_timing_dual/0001

4. Reset/start Board A actor if needed after the Board B listener is active.

Result classification:
PASS means the external observer measured Board A's external trigger-to-ack timing path through PB8/PB9 under the existing gate.

FAIL with missed_ack_count=0, unexpected_ack_count=0, and capture_error_count=0
means the external observer produced honest timing evidence, but the observed
path did not satisfy the gate.

Timeout means the observer did not see 10,000 PB8 trigger edges.

Nonzero missed_ack_count, unexpected_ack_count, or capture_error_count is an
integrity-counter failure and must be retained/classified as bring-up failure,
not timing success.
