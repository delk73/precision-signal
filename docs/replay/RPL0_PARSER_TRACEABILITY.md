# RPL0 Parser Traceability Matrix

Purpose: provide audit-grade traceability from normative RPL0 v1 parser rules to:
- parser enforcement points in [scripts/inspect_artifact.py](../../scripts/inspect_artifact.py)
- acceptance/rejection/mutation tests
- CI gates that execute those tests

Primary normative spec source: [docs/spec/rpl0_artifact_contract.md](../spec/rpl0_artifact_contract.md).

## Key Constants

- `FRAME_SIZE = 16` bytes
- `V1_HEADER_MIN = 0x98` bytes

## Rule Inventory

Rule IDs in this document are stable and versioned as `RPL0-P-###`.

- Dispatch rules: magic, frozen v0/v1 dispatch, unsupported-version reject.
- Header structural rules: `header_len` bounds, `frame_size` invariance, policy fields.
- Schema rules: schema boundary and schema hash integrity.
- Frame region rules: computed frame region length and truncation rejection.
- Canonical identity rules: canonical length, strict trailing behavior, hash-mode trailing tolerance.
- Opaque metadata rules: metadata fields are accepted as opaque bytes.

## CI Parser Gates

Parser traceability evidence is executed in CI via:
- `Replay Artifact Parser Adversarial Tests` (`python3 scripts/test_artifact_parser_adversarial.py`)
- `Replay Artifact Parser Valid v1 Coverage` (`python3 scripts/test_artifact_parser_valid_v1.py`)
- `Replay Artifact Parser Mutation Corpus` (`python3 scripts/test_artifact_parser_mutation_corpus.py`)

These steps are defined in `.github/workflows/ci.yml`.

## Notes

- `flags == 0` and `reserved == 0` are currently enforced by parser policy (`REQUIRED_FLAGS_V1`, `REQUIRED_RESERVED_V1`) and validated by tests; this is stronger than the base v1 spec language and should remain synchronized with any future contract amendment.
- The matrix intentionally references executable evidence only (parser code, dedicated parser tests, and CI parser gate steps).
