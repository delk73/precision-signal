# Persistent Divergence Localization Demo (V2)

## 1. Purpose of Demo V2

Demo V2 proves that replay tooling can localize the first frame of a
persistent divergence artifact pair.

Committed fixture provenance:

- `artifacts/demo_persistent/run_A.rpl` is canonical.
- `artifacts/demo_persistent/run_B.rpl` is a constructed fixture generated by
  deterministic post-processing (`sample += 1` from frame `4096` onward).

## 2. Why Demo V1 is insufficient for this claim

Demo V1 ([docs/demos/demo_capture_perturbation.md](demo_capture_perturbation.md)) perturbs output for exactly one
frame (`sample += 1` at frame `4096`). It demonstrates first-difference
localization, but reconverges at frame `4097`, so it does not demonstrate
persistent trajectory divergence.

## 3. Perturbation mechanism and provenance

Committed fixture mechanism:

- `run_B` is constructed from `run_A` by applying a persistent sample shift from
  frame `4096` onward.

Optional capture workflow (not the provenance of committed fixture):

Firmware feature `demo-persistent-divergence` applies one state perturbation
inside the ISR:

```rust
if frame_idx == 4096 {
    phase_accumulator += STEP;
}
```

In that firmware path, `STEP` is one output-step increment in the phase domain
(`0x0100_0000`), so the sample trajectory remains shifted after frame `4096`.

## 4. Divergence frame

First divergence frame is exactly `4096`.

## 5. Validation commands

```bash
# Validate committed fixture pair
python3 scripts/artifact_tool.py verify artifacts/demo_persistent/run_A.rpl --signal-model phase8
python3 scripts/artifact_tool.py verify artifacts/demo_persistent/run_B.rpl --signal-model phase8
python3 scripts/artifact_tool.py verify artifacts/demo_persistent/run_B.rpl --signal-model none

# Optional: capture provenance for execution-state divergence (hardware path)
make FW_FEATURES= flash-ur
SERIAL=/dev/ttyACM0 python3 scripts/repeat_capture.py \
  --contract rpl0 --runs 1 --signal-model phase8 --reset-mode stlink \
  --artifacts-dir artifacts/demo_persistent/a_capture
cp artifacts/demo_persistent/a_capture/run_01.bin artifacts/demo_persistent/run_A_captured.rpl

make FW_FEATURES=demo-persistent-divergence flash-ur
SERIAL=/dev/ttyACM0 python3 scripts/repeat_capture.py \
  --contract rpl0 --runs 1 --signal-model phase8 --reset-mode stlink \
  --artifacts-dir artifacts/demo_persistent/b_capture
cp artifacts/demo_persistent/b_capture/run_01.bin artifacts/demo_persistent/run_B_captured.rpl
```

Local captured evidence status:

- Captured evidence for the `demo-persistent-divergence` path has been obtained
  locally and is represented by the committed files:
  - `artifacts/demo_persistent/run_A_captured.rpl`
  - `artifacts/demo_persistent/run_B_captured.rpl`
- Verification transcripts are retained in local/archive demo runlogs named
  `demo_v2_release_*.txt` (the [docs/audits/runlogs/](../audits/runlogs/) directory is an ignored
  payload surface and is not a committed proof surface).
- Distinction remains explicit:
  - committed fixtures (`run_A.rpl`, `run_B.rpl`) are the canonical Demo V2 proof
    artifacts;
  - captured artifacts are local regenerated evidence from hardware workflow.

Captured-artifact policy in this repository:

- `run_A_captured.rpl` / `run_B_captured.rpl` are repo-committed local evidence
  artifacts for reproducible operator review.
- They are not normative release-hash anchors; release closeout uses
  tagged/archive runlogs plus audit records as the publication evidence surface.

Expected interpretation:

- `run_A --signal-model phase8` -> PASS
- `run_B --signal-model phase8` -> FAIL at frame `4096` (expected)
- `run_B --signal-model none` -> PASS

## 7. Frame inspection example

```bash
python3 scripts/artifact_tool.py inspect artifacts/demo_persistent/run_A.rpl --frames 4093:4100
python3 scripts/artifact_tool.py inspect artifacts/demo_persistent/run_B.rpl --frames 4093:4100
```

Expected window:

```text
Frame   run_A      run_B
4093    equal      equal
4094    equal      equal
4095    equal      equal
4096    0x00000000 0x00000001
4097    0x00000001 0x00000002
4098    0x00000002 0x00000003
4099    0x00000003 0x00000004
```

Divergence remains persistent after frame `4096`.

## 8. Claim boundaries

This demo supports the claim:

> A persistent-divergence fixture pair can be compared to deterministically
> identify the first divergence frame.

If fixtures are recaptured via `demo-persistent-divergence`, an execution-state
divergence claim can be made for that captured evidence.

This demo does not claim nondeterminism detection. The perturbation is
intentional and deterministic.
