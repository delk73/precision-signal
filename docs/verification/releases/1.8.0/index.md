# Release Evidence Bundle (1.8.0)

This directory is the retained release record for minor release `1.8.0`.

## Release Summary

Release `1.8.0` is a firmware-including release restoring the RPL0 primary
capture path, formalizing `precision.meta.v2`, adding `dpw4` streaming SHA-256
helpers, and separating the timing characterization fixture into a dedicated
crate (`replay-fw-f446-timing`).

## Release Classification

**Firmware-including.** This bundle requires both the standard software-gate
evidence files and the RPL0 firmware capture evidence generated by
`make fw-gate` + `make fw-release-archive VERSION=1.8.0`.

## Scope of this cut

- RPL0 primary firmware capture path restored in `crates/replay-fw-f446`
- `precision.meta.v2` formalized as the authoritative metadata contract
- `dpw4` canonical streaming payload SHA-256 helpers (`OriginHeader`, `compute_payload_hash`, `compute_stream_hash`)
- `crates/replay-fw-f446-timing` separated as a standalone characterization crate
- `sig-util validate` scoped as toolchain-only gate (artifact-contract awareness deferred)
- retained release evidence for `1.8.0`

## Claim boundaries

- primary precision CLI surface (`crates/dpw4`) and RPL0 firmware capture path (`crates/replay-fw-f446`)
- Tier-1 Kani evidence required; Tier-2/Tier-3 optional unless separately retained
- `sig-util validate` is toolchain-only in this release — does not inspect `meta.json` schema
- `precision.meta.v2` enforcement is within the `precision` binary at record and replay time
